## 1. ВВЕДЕНИЕ В GIT

Git — это распределенная система контроля версий. В отличие от централизованных систем (Subversion, TFS), у каждого разработчика на компьютере хранится полная копия всей истории проекта.

**Ключевые термины:**
- **Репозиторий (Repo):** Хранилище кода и истории изменений.
- **Коммит (Commit):** «Снимок» состояния файлов в определенный момент времени. Имеет уникальный хеш (например, `a1b2c3d`).
- **Ветка (Branch):** Изолированная линия разработки. По умолчанию обычно `main` или `master`.
- **HEAD:** Указатель на текущий коммит (где вы находитесь сейчас).
- **Индекс (Staging Area):** Промежуточная зона, куда файлы попадают перед коммитом.
